## 1. 总体架构

本项目是一个 Telegram 运营管理平台，当前以单仓库承载前端、后端、worker、数据库迁移和部署脚本。运行形态是 React/Vite 前端调用 FastAPI API，FastAPI 通过 SQLAlchemy/PostgreSQL 持久化业务状态，并通过任务中心 worker 与 Telegram Gateway 执行真实 TG 动作。

核心链路：

```text
前端视图/弹窗
  -> frontend/src/shared/api/client.ts
  -> frontend/src/app/context/*.ts action 模块
  -> backend/app/api/routers/*.py
  -> backend/app/services/**/*.py
  -> backend/app/models/*.py + PostgreSQL
  -> backend/app/integrations/telegram/* 或 backend/app/ai_gateway.py
  -> 执行结果 / 审计 / runtime summary / issue
  -> 前端按页面刷新或详情下钻
```
