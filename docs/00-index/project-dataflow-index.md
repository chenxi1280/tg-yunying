## 2. 通用请求流

```text
用户页面/弹窗操作
  -> frontend/src/app/views/* 或 context/*Actions.ts
  -> frontend/src/shared/api/client.ts: api/apiWithMeta
  -> FastAPI permission_middleware
  -> backend/app/api/routers/<domain>.py:<handler>
  -> backend/app/services/** 业务规则
  -> backend/app/models/** + PostgreSQL
  -> 可选：worker / Telegram Gateway / AI Gateway / Redis / 文件存储
  -> API 响应 / ApiError.detail
  -> 前端 action/view 更新状态或展示错误
```

## 6. 维护口径

- 新增、删除或改名 API 路由时，必须同步更新本文件对应流转记录，并核对前端入口、service、数据落点和测试入口。
- 改 PRD 中的页面数据加载、错误暴露、任务执行、账号登录、准入、风控或素材逻辑时，先在本索引找到对应 `DF-*` 或 `BG-*`，再进入代码文件。
- 如果一条业务动作同时写主表、runtime summary、operation issue、audit log，记录中优先写主落点；细节以 service 和 migration 为准。
- 静态扫描无法完全展开动态路径和深层 service 调用；遇到动态路径时以前端业务域入口和 router handler 为起点继续追踪。
