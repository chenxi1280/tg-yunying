# 频道任务关注前置阶段设计

## 1. 背景

频道浏览、点赞、评论和回复任务的真实执行前提是账号已经能够访问目标频道。当前只按全局可用账号和频道目标做预检，会出现两类问题：

- 所有账号都没有关注频道时，目标频道无法进入可执行状态，任务创建和启动链路不清晰。
- 只有部分账号关注频道时，任务容易只使用已关注账号，无法自动补齐账号范围内其他账号的关注动作。

新的产品口径是：运营人员创建的是频道互动任务，系统自动在启动前补齐账号关注频道的前置阶段。

## 2. 目标

- 支持通过 `@username`、公开频道链接、邀请链接或已知 peer id 手动添加频道目标。
- 邀请链接包含用户从 Telegram 频道“加入/邀请”入口复制的 `https://t.me/+...`、`https://t.me/joinchat/...`、`telegram.me/+...` 等链接。
- 频道目标可以先进入运营目标库，不要求所有账号已经关注。
- 启动频道浏览、点赞、评论或回复任务时，未授权或不可发送的频道自动让任务账号范围内的账号先关注频道；已授权且可发送的频道可跳过关注前置阶段。
- 关注频道动作必须按随机顺序、抖动、批次和限速执行，避免账号集中操作。
- 主互动阶段只使用已确认关注成功或原本已关注的账号。
- 频道评论 / 回复在运行时再次发现账号未关注、未加入或无法进入关联讨论区时，必须补齐准入动作并延后当前评论，而不是直接记失败或跳过。
- 任务详情能追踪每个账号的前置关注状态、失败原因、重试状态和对主任务容量的影响。

## 3. 非目标

- 不把“关注频道”做成运营人员必须单独创建的普通任务。
- 不新增独立前端业务功能或新的任务类型入口；现有任务创建、预检和任务详情只消费后端返回的阶段状态。
- 不承诺 peer id 一定可以完成关注。peer id 适合识别已同步目标；需要主动加入时优先使用 `@username`、公开链接或邀请链接。

## 4. 业务流程

```text
创建频道互动任务
  -> 选择或手动添加频道目标
  -> 选择账号范围
  -> 创建前预检
  -> 保存草稿或创建并启动
  -> 解析频道目标
  -> 判断频道是否已授权且可发送
  -> 已授权且可发送：直接进入浏览 / 点赞 / 评论 / 回复主互动阶段
  -> 未授权 / 不可发送：生成 ensure_channel_membership 前置动作
  -> 未授权 / 不可发送：按抖动和限速执行关注频道
  -> 未授权 / 不可发送：汇总关注结果
  -> 未授权 / 不可发送：已关注账号进入浏览 / 点赞 / 评论 / 回复主互动阶段
  -> 任务详情持续展示前置阶段和主阶段结果
```

## 5. 频道目标录入

手动添加频道时允许以下输入：

| 输入 | 用途 | 风险 |
| --- | --- | --- |
| `@channel_name` | 公开频道解析和关注 | 频道改名后需要重新同步 |
| `https://t.me/channel_name` | 公开频道解析和关注 | 同上 |
| `https://t.me/+...` / `https://t.me/joinchat/...` | 私有频道或邀请入口加入 | 链接可能过期、撤销或有加入限制 |
| peer id | 已同步目标识别 | 通常不能单独完成主动关注 |

目标进入系统后可以是“待确认”状态。任务允许引用待确认目标，但启动前必须由可用账号解析并确认目标可访问。

录入识别规则：

- 公开频道链接会规范为 username，例如 `https://t.me/example_channel` 规范为 `example_channel`。
- 邀请链接会保留为可加入链接，执行前置关注时用邀请 hash 加入。
- `@username` 会去掉 `@` 后保存。
- peer id 会原样保存；如果账号本来没有访问权限，peer id 不能替代邀请链接完成主动加入。

## 6. 前置动作设计

新增系统动作类型：

```text
ensure_channel_membership
```

动作含义：

- 对指定账号确认是否已关注目标频道。
- 如果已经关注，记录 `already_joined`，不重复加入。
- 如果未关注，尝试加入频道。
- 加入成功后记录 `joined`，并更新账号与频道目标的关联能力。
- 加入失败时记录失败类型、失败详情和是否可重试。
- 如果任务账号范围内一个账号都没有关注频道，会先为全部可用账号生成关注动作；如果只有部分账号已关注，只为未关注账号补关注动作，已关注账号直接进入 `already_joined`。

前置动作属于频道互动任务的内部阶段，不是用户侧新增任务类型。

已授权且可发送的频道不需要生成前置关注动作，可以直接进入主互动规划；运行时仍保留账号、容量、目标能力和风控校验。

频道评论 / 回复的运行时准入守卫必须比浏览 / 点赞更细：

- 如果账号没有账号-频道关系，或 Telegram 返回明确的 `not participant` / 未加入 / 未关注类错误，当前评论 action 进入等待准入，系统补齐 `ensure_target_membership`，准入成功后再重试评论。
- 如果账号已确认关注 / 加入，但 Telegram 仍拒绝评论，记录该账号对当前频道评论区不可发言，并跳过该账号后续评论；其他账号不受影响。
- 如果 Telegram 明确表示该频道消息没有讨论区、不是频道帖子、评论入口不可解析或评论关闭，记录消息级不可评论，后续同帖评论直接跳过。
- 其他错误保留原始失败码和详情，不得改写成“群无权限”“评论区不可用”等泛化文案。

## 7. 抖动和限速

关注频道不能集中瞬时执行，默认策略：

- 账号执行顺序随机打散。
- 单账号之间加入随机间隔，例如 30-180 秒。
- 大批量账号分批执行，例如每批 10-20 个账号。
- 同一频道有全局加入速率限制，避免短时间内大量账号同时关注。
- 遇到 FloodWait 时只延后对应账号，不阻塞整个任务。
- 账号处于受限、离线、未登录或代理异常时直接进入等待或跳过，不强制执行。

具体数值应由系统设置或风控中心提供默认值，任务高级设置只在必要时暴露覆盖项。

## 8. 状态流转

频道互动任务增加前置阶段状态：

| 状态 | 含义 |
| --- | --- |
| `membership_pending` | 等待生成或执行关注频道动作 |
| `membership_running` | 正在按抖动节奏执行关注频道 |
| `membership_partial` | 部分账号已满足关注条件，部分失败或等待 |
| `membership_ready` | 至少一个账号可进入主互动阶段 |
| `membership_blocked` | 没有任何账号可关注或已关注，主互动阶段不能启动 |

主任务状态仍沿用任务中心现有状态。阶段状态放入任务统计、阶段摘要或详情字段中。

## 9. 主互动阶段准入

浏览、点赞、评论和回复动作规划时必须使用目标频道过滤账号：

- 已关注或刚刚关注成功。
- 账号状态为可用。
- 账号容量未超限。
- 目标频道能力满足当前动作类型。
- 评论 / 回复还必须满足账号可以访问评论讨论区；没有准入关系的账号不得直接规划评论，应先生成关注 / 加入前置动作。

如果部分账号关注失败，默认策略是：

- 使用已关注成功账号继续执行主互动阶段。
- 任务详情展示容量缺口。
- 失败账号进入可重试队列或等待人工处理。

如果 0 个账号满足关注条件：

- 主互动阶段不能开始。
- 任务进入异常或等待处理状态。
- 错误原因明确显示为“没有账号成功关注目标频道”。

## 10. 预检输出

任务创建和启动前预检需要补充：

- 目标频道解析状态。
- 账号范围内已关注账号数。
- 需要关注账号数。
- 不可关注账号数和原因。
- 预计关注动作量。
- 预计前置阶段耗时区间。
- 关注阶段是否会造成主互动容量缺口。

预检只用于提示和减少无效提交。Worker 启动前和每次规划动作前仍必须重新读取账号、目标和风控状态。

## 11. 异常处理

| 场景 | 处理 |
| --- | --- |
| 邀请链接失效 | 前置阶段阻塞，提示更换链接 |
| 账号已经关注 | 标记 `already_joined` |
| 账号被限制加入频道 | 记录账号受限，跳过或等待处理 |
| FloodWait | 单账号延后到可执行时间 |
| 目标不可访问 | 阻塞主互动阶段 |
| 评论运行时发现账号未关注 / 未加入 | 补齐前置关注动作，当前评论延后重试 |
| 评论运行时发现账号已准入但不可评论 | 标记账号级评论权限异常，只跳过该账号后续评论 |
| 评论运行时发现消息本身不可评论 | 标记消息不可评论，同帖评论跳过并展示“该消息无法评论” |
| 评论运行时出现未知 TG/API 错误 | 保留原始失败码、详情和尝试记录 |
| 部分账号失败 | 成功账号继续主互动，失败账号可重试 |
| 全部账号失败 | 主互动阶段不启动 |

## 12. 后端实现范围

建议落点：

- 运营目标：手动添加频道目标时保存原始输入、解析状态和可加入凭据。
- 任务中心预检：补充频道 membership 能力摘要。
- Planner：频道互动任务启动后先生成 `ensure_channel_membership` 前置动作。
- Dispatcher：执行关注频道动作，写入账号与频道目标关联状态。
- 账号选择：频道主互动动作只选择已关注目标频道的账号。
- 任务详情：展示前置阶段进度、失败原因和容量缺口。

## 13. 验收标准

- 所有账号都未关注频道时，可以创建频道互动任务；启动后先执行关注频道前置阶段。
- 部分账号已关注频道时，已关注账号跳过前置关注，未关注账号按抖动执行关注。
- 已授权且可发送的频道可以跳过关注前置阶段，直接规划浏览、点赞、评论或回复动作。
- 前置阶段 0 个账号成功时，浏览、点赞、评论和回复动作不会被规划。
- 前置阶段部分成功时，主互动阶段只使用成功账号，并展示容量缺口。
- 关注动作具备随机顺序、抖动间隔、批次限速和 FloodWait 单账号延后。
- 任务详情能看到每个账号的关注状态、失败原因、重试结果和主互动执行结果。
- 频道评论运行时遇到未关注 / 未加入账号时，会补齐准入并延后评论；准入成功后该账号可以继续评论。
- 已准入账号被 Telegram 拒绝评论时，只标记该账号对频道评论区不可发言，不影响同帖其他账号继续评论。
- 频道消息本身不可评论时，详情明确显示“该消息无法评论”，并且同帖后续评论不会继续重试。
- 其他未归类错误在详情中保留原始失败码和原始返回摘要，便于查看尝试记录和 Trace。
