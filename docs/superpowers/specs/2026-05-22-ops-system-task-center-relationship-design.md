# 系统配置、运营中心、任务中心关系设计

## 设计目标

把平台主线从“运营人员到任务中心手工建任务”调整为“运营中心发现问题、维护方案、生成任务，任务中心负责执行明细”。三者边界固定为：

```text
系统配置 = 平台底座能力
运营中心 = 日常工作台 + 运营方案 + 异常处理入口
任务中心 = 执行详情 / 失败事实源 / 调度控制台
```

## 模块边界

### 系统配置

系统配置只回答“平台能不能跑”。它由管理员维护，不承载具体运营策略。

- Telegram 开发者应用、账号登录基础能力。
- AI 供应商、默认模型、token、健康检查和回退策略。
- 素材基础能力、缓存设置和上传能力。
- 后台账号、权限、额度和全局安全默认值。
- 全局运行开关、worker / 队列 / 代理等基础设置。

系统配置不编辑群活跃话题、频道互动节奏、AI 暖场策略、转发规则或任务失败处理流程。

### 运营中心

运营中心是运营人员默认入口，上半部分是工作台，下半部分是运营方案 / 策略模板。

工作台默认按运营目标展示：

- 哪些群、频道、讨论组变冷、异常或有机会。
- 哪些任务失败影响了这个目标。
- 哪些账号、目标权限、规则、AI 质量、风控或 TG 限制需要处理。
- 每个异常的影响范围、严重程度、建议动作和入口。

运营方案负责长期策略：

- 群活跃方案：接话优先、低频暖场、话题种子、AI 账号角色、没人接时降频。
- 频道互动方案：浏览、点赞、评论、回复的比例、节奏和目标消息范围。
- 转发监听方案：源群、目标群、过滤、转换、路由和目标群发送量。
- 账号运营方案：账号分组、冷却、养号、安全动作和恢复策略。

运营方案可以生成或调整任务；运营中心不展示每条 action 原始明细，只展示能指导运营判断的聚合结果。

### 任务中心

任务中心是执行层工作台，不再是普通运营人员的默认入口。

- 记录 Task、Action、ExecutionAttempt、准入子任务和失败原始结果。
- 展示任务详情、action 明细、失败码、失败堆栈、重试记录、计划时间和执行账号。
- 支持启动、暂停、继续、停止、重试、重置、删除和高级手动创建。
- 每个任务必须能追溯到来源方案、运营目标、规则版本、账号范围和执行结果。

任务中心负责记录失败，运营中心负责暴露失败、归因失败、引导处理失败。

## 运营异常模型

运营中心需要统一的“运营异常”概念。第一版可以从任务和 action 实时聚合，不要求立即建表；产品口径先固定。

```text
operation_issue
- target_id
- issue_type
- severity
- source_task_ids
- source_action_ids
- affected_account_ids
- failure_type
- failure_reason
- suggested_action
- status: open / acknowledged / resolved / ignored
```

默认展示粒度：

```text
按运营目标聚合
  -> 展开目标异常
    -> 查看关联任务失败
      -> 跳转任务详情 / 账号处理 / 目标处理 / 规则处理
```

运营中心只展示“需要运营判断或影响目标效果”的失败，不把每条 action 原样铺出来。任务中心保留完整执行明细。

## 失败上卷规则

任务中心是失败事实源：

- action.result、execution_attempts、task.stats 和 task.last_error 记录原始失败。
- 失败必须保留 task_id、action_id、account_id、target_id、failure_type、时间和可读原因。

运营中心是失败处理入口：

- 按 target_id 聚合异常。
- 同一目标下按 issue_type、severity、failure_type 和 source_task_ids 聚合。
- 默认展示影响最大的目标异常，再展示关联任务明细。
- 每个异常给出建议动作，例如重新登录账号、重新准入目标、检查频道讨论组、暂停任务、查看规则、调整运营方案。

## 数据流

```text
系统配置提供基础能力
        ↓
运营中心维护目标方案和策略模板
        ↓
方案生成或调整任务
        ↓
任务中心执行 Task / Action
        ↓
失败事实写入任务中心
        ↓
运营异常聚合回流运营中心
        ↓
运营人员按目标处理异常或调整方案
```

## 验收口径

- 运营人员日常入口是运营中心，不需要先去任务中心查失败。
- 运营中心默认按目标展示异常，点开后看到关联任务失败。
- 任务中心保留完整执行详情和失败事实，不把策略模板做成任务中心主功能。
- 系统配置不承载具体运营策略，只提供平台底座能力。
- AI 暖场、降频、话题种子、账号角色等属于运营方案；AI 供应商和默认模型属于系统配置；某一轮 AI 为什么跳过属于任务中心详情，并上卷为运营中心异常。
