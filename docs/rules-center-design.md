# 规则中心模块设计文档

## 1. 模块定位

规则中心是 TG 运营管理平台的统一规则配置中心，负责管理自动化任务执行过程中的输入过滤、输出校验、内容转换、素材选择、路由分发、账号策略、限速策略、重试策略、规则测试、命中记录和版本回滚。

系统不再把“关键词库”作为独立业务模块展示。关键词、敏感词、白名单词、路由词都作为规则集里的条件存在，由规则集统一管理、统一发布、统一测试。

核心原则：

```text
系统统一维护规则集
任务绑定规则集版本
关键词只是规则条件
规则发布后形成固定版本
任务执行按绑定版本处理
```

## 2. 核心概念

| 概念 | 说明 |
| --- | --- |
| 规则集 | 一套完整的任务处理规则，例如“转发监听通用规则”“AI 回复安全规则”“评论发送规则”。 |
| 规则版本 | 规则集的一次发布快照。任务实际执行时绑定具体版本或跟随当前发布版本。 |
| 输入过滤 | 判断外部输入内容是否允许进入后续流程。 |
| 输出校验 | 判断系统即将发送的内容是否合规。 |
| 内容转换 | 对内容做清洗、替换、去链接、去风险词、裁剪等处理。 |
| 素材选择 | 根据任务类型、消息类型、标签、关键词、目标和频控选择图片、表情包或组合消息素材。 |
| 路由规则 | 决定内容发到哪些目标群、频道或评论区。 |
| 账号策略 | 决定使用哪个 TG 账号执行发送、评论或回复。 |
| 限速策略 | 控制发送频率、账号冷却、目标冷却和每日上限。 |
| 重试策略 | 定义失败后的重试次数、间隔、换号和暂停策略。 |

## 3. 模块结构

```text
规则中心
  ├─ 规则集管理
  ├─ 规则版本管理
  ├─ 输入过滤规则
  ├─ 输出校验规则
  ├─ 内容转换规则
  ├─ 素材选择规则
  ├─ 路由分发规则
  ├─ 账号策略
  ├─ 限速策略
  ├─ 重试策略
  ├─ 规则测试器
  ├─ 命中记录
  └─ 审计与回滚
```

## 4. 任务绑定方式

所有自动化任务都不再单独选择关键词库，而是选择规则集和规则版本。

```text
创建任务
  ↓
选择任务类型
  ↓
选择规则集
  ↓
选择规则版本，或使用当前发布版本
  ↓
任务启动
  ↓
执行时按规则版本处理
```

| 任务类型 | 是否需要规则集 | 主要使用规则 |
| --- | --- | --- |
| 监听转发 | 必须 | 输入过滤、内容转换、路由、账号、限速、重试 |
| AI 回复 | 必须 | 输入过滤、AI 输出校验、内容转换、素材选择、账号、限速 |
| AI 评论发送 | 必须 | 输入过滤、AI 输出校验、内容转换、账号、限速、重试 |
| 普通消息发送 | 建议 | 输出校验、素材校验、账号、限速 |
| 风控类任务 | 必须 | 输入过滤、输出校验、拦截 |

## 5. 监听转发执行流程

监听转发必须在入口直接过滤。风险消息不进入转换、路由和发送流程。

```text
源群新消息
  ↓
输入过滤规则
  ↓
不通过：跳过 / 拦截，记录原因
  ↓
通过：内容转换
  ↓
素材处理：保留源媒体 / 选择素材 / 丢弃媒体 / 降级文本
  ↓
路由分发
  ↓
账号选择
  ↓
限速检查
  ↓
发送
  ↓
失败重试 / 记录结果
```

监听转发规则重点：

- 黑名单关键词
- 白名单关键词
- 链接规则
- 媒体类型规则
- 发送人规则
- 消息长度规则
- 内容转换规则
- 源媒体保留规则
- 素材库图片 / 表情包替换规则
- 源群到目标群映射
- 关键词路由
- 目标群账号策略
- 发送限速
- 失败重试

## 6. AI 回复执行流程

AI 回复先对输入消息和上下文做过滤，通过后再生成候选内容，并对候选内容逐条做输出校验。

```text
用户消息 / 群上下文
  ↓
输入过滤规则
  ↓
不通过：不处理，不进入 AI
  ↓
通过：AI 生成多条候选回复
  ↓
逐条输出校验
  ↓
不通过：转换后再校验 / 丢弃该条
  ↓
通过：按规则选择可选素材后发送
```

AI 一次生成多条候选时，校验必须逐条处理，不因为一条失败而整批废弃。

```text
AI 返回 5 条
- 第 1 条通过：发送
- 第 2 条命中风险词：转换后再校验
- 第 3 条仍不通过：丢弃
- 第 4 条通过：发送
- 第 5 条命中链接规则：丢弃
```

推荐默认策略：

```text
AI 回复：转换一次，仍失败则丢弃
AI 表情包：AI 只输出素材意图，规则引擎选择具体素材
```

## 7. AI 评论发送流程

评论发送本质也是 AI 生成内容后发送，因此和 AI 回复共用输入过滤、输出校验、转换和逐条处理逻辑。

```text
频道消息 / 评论对象 / 评论上下文
  ↓
输入过滤规则
  ↓
不通过：不处理，不进入 AI
  ↓
AI 生成多条评论候选
  ↓
逐条输出校验
  ↓
不通过：转换后再校验 / 丢弃该条
  ↓
通过：发送评论
  ↓
失败重试 / 记录结果
```

推荐默认策略：

```text
AI 评论：转换一次，仍失败则丢弃
```

## 8. 规则动作设计

输入过滤动作：

| 动作 | 含义 |
| --- | --- |
| 允许 | 进入后续流程。 |
| 跳过 | 不处理，记录跳过原因。 |
| 拦截 | 不处理，记录风险原因。 |
| 标记 | 继续处理，但打风险标签。 |

输出校验动作：

| 动作 | 含义 |
| --- | --- |
| 通过 | 允许发送。 |
| 转换 | 清洗内容后再次校验。 |
| 丢弃 | 放弃当前内容。 |
| 重写 | 重新生成内容。 |
| 拦截 | 禁止发送并记录风险。 |

内容转换动作：

- 删除关键词
- 替换关键词
- 删除链接
- 替换链接
- 删除 @ 提及
- 删除来源署名
- 删除敏感表达
- 裁剪超长文本
- 添加前缀
- 添加尾注

素材动作：

- 保留源媒体
- 丢弃源媒体
- 按标签选择素材
- 指定固定素材
- 随机选择表情包
- 替换为素材库图片
- 追加图片或表情包
- 素材失败时降级文本
- 素材失败时换素材
- 素材失败时跳过

## 9. 规则版本机制

规则集必须版本化。

```text
编辑规则
  ↓
保存为草稿
  ↓
规则测试
  ↓
发布版本
  ↓
任务绑定版本
  ↓
执行记录关联版本
```

版本规则：

- 每次发布生成新版本。
- 已发布版本不可直接修改。
- 修改规则必须生成新版本。
- 任务可以绑定固定版本。
- 任务也可以选择跟随当前发布版本。
- 规则异常时可以回滚到历史版本。

规则版本解析规则：

- 绑定固定版本的任务，执行项生成时直接使用绑定版本。
- 跟随当前发布版本的任务，执行项生成时解析当时的当前发布版本。
- 无论使用固定版本还是跟随当前发布版本，每个执行项生成时都必须固化 `resolved_rule_set_version_id`。
- 执行项、发送记录、命中记录和审计记录永远追溯到具体规则版本，不只记录“当前发布版本”。
- 规则发布后，只影响之后新生成的执行项，不回写已经生成的执行项。
- 规则版本中的素材选择策略必须追溯到素材条件、候选素材范围和选择动作；具体执行项生成时还要固化最终 `material_id`、`asset_version_id` 和素材指纹。
- 已发布规则版本引用过的素材不能物理删除，只能停用或软删除；素材停用后，后续执行项按规则换素材、降级文本或跳过，历史执行记录不回写。

推荐默认：

```text
重要任务绑定固定版本
普通任务可跟随当前发布版本
```

## 10. 规则测试器

规则测试器按任务类型切换，分别模拟监听转发、AI 回复、AI 评论和普通发送。

测试模式：

| 模式 | 说明 |
| --- | --- |
| 仅规则测试 | 不调用 AI，不创建执行项，不发送，只对输入样例和输出样例做规则判断。 |
| AI 干跑测试 | 可以调用 AI 生成候选内容，但不发送、不创建真实执行项，只展示校验、转换和丢弃结果。 |
| 历史样本回放 | 使用历史源消息、历史 AI 输出或历史评论场景回放规则效果，用于验证新版本是否会改变处理结果。 |

所有测试模式都必须标记为测试数据，不写入真实发送队列，不影响账号限额和任务进度。

媒体缓存场景必须能模拟：

- 源媒体待缓存，执行项进入 `waiting_material_cache`。
- 等待超时后缓存才完成，验证不会再补发。
- `source_media_cached` 旧事件迟到，验证不会唤醒当前执行项。
- 相册其中一张图缓存或发送失败，验证剔除失败图后继续保持剩余图片顺序。
- 等待队列超过容量，验证直接放弃受影响素材缓存并记录队列保护原因。

监听转发测试：

```text
输入源消息
选择规则集版本
选择源群
系统展示：
- 是否通过输入过滤
- 命中哪些规则
- 转换后内容
- 路由目标
- 账号策略
- 限速判断
```

AI 回复测试：

```text
输入用户消息 / 群上下文
选择规则集版本
系统展示：
- 输入是否通过
- AI 候选内容
- 每条候选是否通过输出校验
- 不通过原因
- 转换后内容
- 最终可发送内容
```

AI 评论测试：

```text
输入评论场景
选择规则集版本
系统展示：
- 生成评论候选
- 每条校验结果
- 转换结果
- 最终发送列表
```

## 11. 前端页面设计

规则中心前端不再展示独立“关键词库”。页面统一围绕规则集、规则版本、规则配置、规则测试和命中记录组织。

```text
规则中心
  ├─ 规则集
  ├─ 规则版本
  ├─ 规则配置
  ├─ 规则测试
  └─ 命中记录
```

### 11.1 规则中心首页

首页用于管理所有规则集。

列表字段：

| 字段 | 说明 |
| --- | --- |
| 规则集名称 | 例如：转发监听通用规则、AI 回复安全规则。 |
| 适用任务 | 监听转发 / AI 回复 / AI 评论 / 普通发送。 |
| 当前版本 | 当前已发布版本，例如 v3。 |
| 绑定任务数 | 当前使用该规则集的任务数量。 |

| 状态 | 启用 / 停用 / 草稿中。 |
| 操作 | 编辑、测试、新建版本、查看任务、回滚。 |

顶部操作：

```text
新建规则集
规则测试
查看命中记录
```

筛选条件：

```text
适用任务类型
规则集状态

关键词搜索
```

### 11.3 输入过滤配置区

输入过滤用于处理外部输入内容，监听转发和 AI 回复都会使用。

字段建议：

```text
黑名单关键词
白名单关键词
链接规则
媒体类型
发送人限制
消息长度
@ 提及规则
正则规则
命中动作
```

命中动作：

```text
跳过
拦截
标记风险
允许继续
```

监听转发默认：

```text
命中高风险规则：直接拦截
未命中白名单：跳过
```

### 11.4 输出校验配置区

输出校验用于 AI 回复、AI 评论、普通消息发送前校验。

字段建议：

```text
禁止关键词
禁止链接
禁止 @
禁止联系方式
禁止引流表达
最大长度
最小长度
输出失败处理策略
```

失败处理策略：

```text
转换一次，仍失败则丢弃
直接丢弃
重新生成一次
固定回复
```

推荐默认：

```text
AI 回复：转换一次，仍失败则丢弃
AI 评论：转换一次，仍失败则丢弃
普通消息：不通过则禁止发送
```

### 11.5 内容转换配置区

内容转换用于把内容清洗后再发送。

转换项：

```text
删除关键词
替换关键词
删除链接
替换链接
删除 @ 提及
删除来源署名
裁剪超长内容
添加前缀
添加尾注
```

转换规则需要支持顺序，因为转换顺序会影响最终结果。

```text
转换顺序：

3. 删除 @
4. 裁剪长度
5. 添加尾注
```

### 11.6 素材选择配置区

素材选择用于普通发送、监听转发和 AI 活跃群中的图片、表情包、文件和组合消息处理。

字段建议：

```text
启用素材规则
适用任务类型
允许媒体类型
允许表情包类型：图片伪表情 / 静态 sticker / animated sticker / video sticker / custom emoji
素材来源：保留源媒体 / 素材库 / 禁用媒体
素材标签
固定素材
随机策略
相册处理：保持原顺序 / 去掉失败图继续发 / 整体跳过 / 失败段降级文本
相册等待策略：ready 分段按顺序发送 / 待缓存分段等待到本轮超时 / 失败分段剔除
素材等待超时：默认到任务本轮超时，建议不超过 30 分钟
超时后补发：禁止；迟到缓存完成事件只记录，不创建补发执行项
等待队列超量：直接放弃受影响素材缓存
源媒体不可恢复：丢弃媒体 / 降级文本 / 换已就绪素材 / 跳过
目标群素材冷却
账号素材冷却
同素材重复冷却
失败处理策略
素材未缓存处理：等待缓存完成 / 丢弃媒体 / 换已就绪素材 / 降级文本 / 跳过
```

监听转发默认：

```text
默认只保留文字
显式开启后才保留源媒体或替换为素材库图片
相册 / media group 默认保持原顺序转发
源媒体待缓存或 FloodWait 时不创建新图，按同一 source_media_asset_id 等待缓存完成、降级或跳过
等待缓存完成默认不超过任务本轮超时；任务暂停时不补发，恢复后再检查
缓存完成事件必须校验版本；旧事件迟到不能唤醒当前执行项
任务时间窗关闭、目标静默期或群上下文过期时不补发
相册默认按 ready 分段原顺序发送，失败图可以剔除后继续发
媒体失败时按规则丢弃媒体、降级文本或跳过；不提供人工缓存
等待队列爆满、不可恢复素材暴增或兜底扫描频繁唤醒时，直接放弃受影响素材缓存
```

AI 活跃群默认：

```text
AI 只输出素材意图
规则引擎按素材标签、目标群、账号角色和频控选择表情包
表情包子类型需按目标、账号和网关能力校验
每轮低频发送，避免连续刷图
```

规则命中记录必须保留：

```text
候选素材范围
最终素材 ID
素材指纹
Telegram 缓存是否就绪
下载重发是否成功
相册组 ID、分段顺序和分段结果
表情包子类型和能力校验结果
等待缓存状态、唤醒来源、超时结果和迟到事件处理结果
缓存事件版本、队列超量、放弃缓存原因和不可恢复原因
失败和回退原因
```

### 11.8 账号策略配置区

字段建议：

```text
账号池
固定账号
轮询
随机
权重随机
目标群粘性
账号冷却
失败后冷却
```

推荐默认：

```text
监听转发：目标群粘性账号策略
AI 回复 / AI 评论：任务账号池轮询 + 冷却限制
```

账号策略和风控中心的边界：

- 规则中心只表达“倾向怎么选账号”，例如账号池、固定账号、轮询、随机、权重随机和目标群粘性。
- 风控中心负责“这个账号现在能不能用、能用多大强度、异常后怎么处置”。
- 规则中心可以给出候选账号顺序或权重，但最终可用账号必须经过风控中心的账号状态、代理健康、限额、冷却和风险评分过滤。
- 如果规则中心选择的固定账号被风控阻塞，执行中心必须按任务失败策略换号、延后、跳过或暂停，而不能绕过风控强行发送。

执行优先级：

```text
账号 / 代理 / 目标硬阻塞
  ↓
风控中心全局硬策略
  ↓
规则中心内容硬拦截
  ↓
风控中心全局限额和冷却
  ↓
规则集限速和重试策略
  ↓
任务级偏好
  ↓
队列级优先级：运行中任务优先于已暂停任务，高优先级任务优先于低优先级任务
  ↓
执行时动态决策
```

也就是说，账号策略是候选和排序，不是强制发送许可。固定账号、目标群粘性和权重随机都不能覆盖账号未登录、Session 失效、代理严重告警、账号 E 级阻塞、目标无权限和内容硬拦截。

```text

```

重试字段：

```text

失败后是否跳过
连续失败是否暂停任务
```

限速与重试边界：

- 规则中心定义任务内的限速和重试偏好，例如单目标冷却、同内容去重、最大重试次数、失败后换号。
- 风控中心定义全局硬边界，例如账号小时 / 日上限、账号全局冷却、代理告警、FloodWait 处置和账号风险等级。
- 执行中心执行时必须先满足风控中心硬边界，再应用规则集限速和重试策略。
- 规则集策略不能放宽全局风控策略，只能在全局允许范围内进一步收紧或细化。

## 12. 旧关键词库废弃策略

旧关键词库不做迁移，不继续作为独立配置入口，也不作为任务可选项保留。

处理原则：

- 新规则中心上线后，所有新任务只使用规则集和规则版本。
- 旧关键词库入口直接下线或隐藏，不再出现在系统设置、任务创建和规则测试器里。
- 旧关键词数据不自动迁入新规则集，避免把旧口径带入新规则体系。
- 如确有需要，运营人员在新规则集中重新配置输入过滤、输出校验或路由条件。
- 文档、页面文案和后续实现都不再使用“关键词库”作为产品概念。

## 13. 最终产品口径

规则中心统一管理所有任务的执行规则。监听转发在入口直接过滤；AI 回复和 AI 评论在生成后逐条做输出校验；不合规内容按规则转换或丢弃；任务只绑定规则集版本，不再单独使用关键词库。
