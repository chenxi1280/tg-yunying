# TG 运营管理平台新版产品设计

## 1. 项目定位

新版 TG 运营管理平台面向 Telegram 运营团队，用一个后台统一管理：

```text
TG 开发者应用
TG 账号
账号分组
群 / 频道 / 联系人资产
运营目标
消息发送
任务中心
监听中心
规则中心
AI 内容生成
归档中心
运营数据
执行记录
审计记录
```

新版不再以多租户 SaaS、卡密订阅、旧 Campaign、AI 草稿人工审核作为主线。平台主线是：

```text
账号接入 -> 资产同步 -> 目标确认 -> 规则配置 -> 任务创建
-> 监听 / AI / Worker 执行 -> 详情追踪 -> 数据复盘 -> 审计留痕
```

## 2. 产品边界

平台只围绕自有、授权、可运营的 TG 账号和目标设计。

- 只能使用已接入账号。
- 只能操作已确认的运营目标。
- 只能监听已授权源群、群聊、频道或讨论组。
- 任务必须有时间窗口、账号池、频控和执行记录。
- 所有发送、监听、过滤、转换、AI 生成、失败、重试都必须留痕。

AI 活跃群和转发监听群不做人工审核。系统通过自动校验、规则过滤、风控、限速、上下文检查和失败重试保证任务可控。

## 3. 功能中心

新版后台按 10 个功能中心组织：

1. 运营概览中心
2. 系统初始化中心
3. TG 账号中心
4. 运营目标中心
5. 消息发送中心
6. 任务中心
7. 监听中心
8. 规则中心
9. AI 内容中心
10. 数据、归档与审计中心

主导航使用：

```text
运营概览
TG账号管理
运营目标
消息发送
任务中心
监听中心
规则中心
归档中心
运营数据
系统设置
审计记录
```

## 4. 运营概览

运营概览回答三个问题：

- 系统现在是否正常？
- 哪些任务正在运行？
- 哪里有风险或失败需要处理？

页面展示账号、目标、任务、监听、AI、Worker、发送成功率、失败记录和风险提醒，并能跳转到异常账号、失败任务、监听异常、发送失败、规则异常和运行中任务详情。

## 5. 系统初始化

系统初始化中心维护前置配置：

- TG 开发者应用
- AI 供应商和模型
- 提示词模板
- 素材管理
- 关键词 / 敏感词规则
- 发送节奏
- 全局风控
- 系统运行配置

没有可用开发者应用时，不能新增 TG 账号或启动账号登录。没有可用 AI 供应商时，AI 内容生成任务应明确失败或使用配置允许的回退策略。

## 6. TG 账号中心

TG 账号中心是执行资源池，负责账号分组、新增账号、验证码/二维码/2FA 登录、健康检查、资料同步、联系人同步、群频道同步、验证码提取、账号详情和账号执行记录。

账号状态包括：

```text
未登录
等待验证码
等待扫码
等待 2FA
在线
离线
异常
需重新登录
冷却中
禁用
```

账号可以配置稳定角色，例如引导型、解答型、补充型、新人型、总结型、公告同步型、活动同步型和备用发送账号。账号可以重复发送，但必须受冷却、上限、目标权限、健康分和行为一致性约束。

## 7. 运营目标中心

账号同步出来的是 TG 资产，运营人员确认后才是运营目标。消息发送、任务中心和归档都只选择运营目标。

目标类型：

- 群聊目标
- 频道目标
- 频道讨论组
- 联系人目标
- 手动目标

目标详情页展示基础信息、账号覆盖、可发言账号、可监听账号、最近消息、历史任务、发送记录、归档记录、风险状态和快捷创建任务入口。

## 8. 消息发送中心

消息发送中心只负责直接发送，不负责复杂自动化。

支持私聊发送、群聊发送、频道发送、批量发送、定时发送、图片素材发送、发送记录、重试、取消和失败详情。

目标来源必须明确标识为联系人、账号可见群、已授权运营目标、频道目标或手动输入目标。手动输入目标需要提示权限和目标有效性风险。

## 9. 任务中心

任务中心是新版核心模块，分三条任务主线：

```text
频道互动任务：
父任务 -> 目标频道消息 -> 动作子任务 -> 账号执行项

AI 活跃群任务：
父任务 -> 活跃循环 Cycle -> 发言 Turn -> 自动校验 -> 账号发送

转发监听群任务：
父任务 -> 源群事件 -> 规则过滤 / 转换 / 路由 -> 转发批次 -> 目标群发送
```

当前实现支持保留默认目标群，同时通过 `target_group_ids` 和规则集版本里的 `routing` 配置扩展为多目标分发；源群映射、关键词路由和默认路由会在 Worker 规划发送项前参与目标群选择。

频道任务详情按频道消息与动作类型形成消息子任务摘要，展示目标数、完成数、失败数、重复数、运行中数量、容量缺口和账号执行项；容量不足不会直接停止任务，但会作为子任务状态暴露给运营人员。

AI 活跃群任务配置支持静默期低频模式、每日爬坡和上下文过期阈值。Worker 规划 Cycle 时会按静默期限制参与账号和每轮发言数，按启动后分钟数逐步提高参与比例；发送执行前如果发现快照之后新增群消息超过阈值，会自动跳过该 Turn 并记录上下文过期原因。

所有任务支持创建、保存草稿、启动、暂停、继续、停止、重试、重置、编辑、删除、查看详情、执行记录、失败原因和审计记录。

统一状态建议：

```text
草稿
已排期
运行中
已暂停
已达目标但未结束
收尾中
已完成
人工停止
异常失败
已删除
```

达到目标数量不等于任务完成；自然结束由 `end_at` 或结束时间控制。

## 10. 频道互动任务

频道互动任务包括频道消息浏览、点赞、评论和回复。

一个频道任务下面，每条频道消息的点赞、评论、回复都是独立子任务。任务详情必须按消息展示目标数、完成数、失败数、重复数、账号执行项、失败原因和容量不足提示。

强去重规则：

```text
同一个账号
同一个频道
同一条消息
同一种动作
只能有效执行一次
```

频道监听采用聚合模式：一个频道只启动一个监听实例，多个任务订阅这个频道事件。

## 11. AI 活跃群任务

AI 活跃群是循环型任务：

```text
父任务 -> 活跃循环 Cycle -> 本轮发言计划 Turn -> 账号发送
```

每轮先拉取群上下文，一次生成多账号对话计划，然后系统自动校验并分配账号按节奏发送。本轮全部发送、取消或过期后才进入下一轮。

自动校验包括空内容、敏感词、重复内容、长度、外链、@ 成员、账号冷却、账号发送上限、目标群频控、上下文过期和 AI 输出格式检查。

校验结果：

```text
通过
跳过
拦截
重生成
延迟发送
```

不再存在待审核、人工审核、审核通过、审核驳回或审核队列。

## 12. 转发监听群任务

转发监听群是事件驱动任务：

```text
源群新消息
-> 监听中心捕获
-> 源消息事件
-> 过滤规则
-> 转换规则
-> 路由规则
-> 账号分配
-> 自动发送到目标群
-> 记录结果
```

过滤、转换、路由、发送账号策略、限速、拦截和重试都应作为系统级规则集配置，任务绑定具体规则版本。

规则处理结果只有：

```text
通过
忽略
拦截
转换失败
发送失败
部分失败
已发送
```

不进入人工审核。

## 13. 监听中心

监听中心从任务中抽离，分为频道监听中心和群监听中心。

- 频道监听服务频道浏览、点赞、评论、回复、新消息任务。
- 群监听服务 AI 活跃群、转发监听群、群归档和群上下文分析。

原则：

```text
一个频道只监听一次。
一个源群只监听一次。
多个任务共享事件。
任务通过订阅获得事件。
```

监听状态页展示监听对象、监听类型、监听账号、备用监听账号、最后事件时间、订阅任务数、事件积压、错误状态和最近错误原因。

## 14. 规则中心

规则中心服务转发监听、AI 内容校验和风控。

包含转发规则集、过滤规则、转换规则、路由规则、关键词规则、敏感词规则、发送策略、限速策略、拦截策略、重试策略、规则测试器和规则版本管理。

每次修改规则应生成新版本，运行中的任务绑定具体规则版本，保证规则变更可追踪、可回滚。

规则中心不使用人工审核口径，统一使用自动处理、过滤、转换、路由、拦截、跳过、限速和重试。

## 15. 数据、归档与审计

运营数据分为账号数据、目标数据、消息发送数据、频道互动数据、AI 活跃群数据、转发监听数据、归档数据、AI 用量、失败与风险数据。

当前实现通过 `/api/operation-metrics/summary` 从账号、运营目标、任务中心执行项、归档和 AI 用量表生成真实汇总，前端运营数据页按上述分类展示，不再只是 AI Token 明细。

归档中心用于沉淀群资产，包括历史消息、成员信息、消息库、成员库、新群初始化方案、JSON 导出、重跑归档和归档审计。

审计必须记录登录、验证码查看、账号新增、账号登录、账号同步、开发者应用修改、AI Key 修改、目标授权、任务创建、任务启动、任务暂停、任务重置、任务停止、任务删除、规则修改、规则版本发布、消息发送、AI 自动生成、自动校验结果、转发规则命中、归档导出和失败重试。

审计页支持按操作人、动作、对象类型、对象 ID、账号 ID、目标 ID、任务 ID、状态、关键词和时间范围筛选，用于回溯一条消息为什么发送、由哪个账号发送、关联哪个任务或规则，以及失败、跳过、重试的原因。

审计回答：

```text
这条消息为什么发？
由哪个账号发？
基于什么上下文生成？
命中了哪些规则？
为什么被跳过？
为什么失败？
是否重试？
```

## 16. 当前落地与兼容边界

当前代码已经具备 FastAPI + React + PostgreSQL 基础、账号登录同步、运营目标、消息发送、任务中心、AI 配置、归档、运营数据和审计页面。

为保证迁移安全，旧 `/api/campaigns`、`/api/ai-drafts`、`/api/operation-tasks` 与 `/api/review-*` 已隔离为显式兼容路由，开发/生产默认不注册；`/api/message-tasks` 继续服务消息发送中心的发送记录、派发、重试和取消。新版页面和后续功能默认围绕 `/api/tasks` 任务中心、运营目标、监听中心和规则中心继续演进。
